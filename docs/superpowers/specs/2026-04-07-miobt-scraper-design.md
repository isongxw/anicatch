# miobt.com 动漫资源爬虫项目设计文档

**日期：** 2026-04-07
**项目类型：** Python Web Scraping
**规模：** 小型单文件脚本

## 1. 项目概述

构建一个简单的 Python 爬虫项目，使用 Scrapling 库抓取 miobt.com 首页的动漫资源信息，包括标题、下载链接、文件大小等数据，并保存为 JSON 格式。

### 目标
- 抓取 miobt.com 首页的动漫资源信息
- 实现错误重试机制
- 添加请求限速和延迟
- 提供清晰的日志输出
- 保存数据为 JSON 格式

### 范围
- 仅抓取首页数据（测试用途）
- 不涉及深层页面爬取
- 不实现增量更新
- 不涉及用户认证

## 4. 功能模块设计

### a) 配置模块
- 请求延迟设置（避免被封）
- 重试次数配置
- 输出路径配置

### b) 抓取模块
- 使用 Scrapling 获取网页内容
- 解析HTML提取数据：
  - 动漫标题
  - 下载链接/磁力链接
  - 文件大小
  - 发布时间
  - 种子数/下载数（如果有）

### c) 错误处理模块
- 网络错误重试机制
- 异常捕获和日志记录

### d) 数据存储模块
- 将数据保存为JSON格式
- 自动创建输出目录

### e) 日志模块
- 显示抓取进度
- 记录成功/失败信息

## 5. 数据流程

```
开始
  ↓
初始化配置和日志
  ↓
发起HTTP请求（带延迟）
  ↓
请求成功？
  ├─ 否 → 重试（最多3次）→ 失败 → 记录错误日志 → 结束
  ↓ 是
解析HTML提取数据
  ↓
保存数据到JSON文件
  ↓
记录成功日志
  ↓
结束
```

### 错误处理策略
- **网络错误**：自动重试3次，每次间隔递增（1s, 2s, 4s）
- **解析错误**：跳过该项，记录错误，继续处理其他项
- **文件写入错误**：抛出异常，程序终止
